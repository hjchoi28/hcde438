MP3 Writeup

Overview & Usage

For MP3, I created a 2D game using the p5.js and p5.play libraries, where the goal is to help the space ktty avoid colliding with asteroids in space. The player can control the movement of the space kitty with their mouse and the game continues infinitely until the player hits the asteroids. However, the speed of the asteroids also increases as the game progresses, making it more difficult. Once the end condition is met, the timer and game resets, and the player is encouraged to try again.

  • Play the game at hjchoi28.github.io/space-kitty/
  • Check out the code in my Public Github Repository
The purpose of MP3 was to learn how to use NPM to install libraries, a local development server, and bundle code. I also learned how to deploy my project on Github Pages.

Development Process

I started this project by setting up my development toolchain using Hannah's example Dev Toolchain Walkthrough. I used the sample p5.play starter code to try deploying my game. Once I confirmed everything worked, I started writing my actual game code.

First, I looked at Allison Parrish's p5.play tutorial to understand the basics of the p5.play library. Then, I divided the core function of my game into manageable steps I could implement. I explored creating sprites by creating a static group of grey square sprites to represent the asteroids, and adding the image of the space kitty to a sprite that follows the mouse position.

After figuring out how the sprites worked, I then worked on making the asteroids dynamic. I struggled with creating a reasonable number of the sprites since the draw() function would continuously create them until it covered the whole screen. I eventually figured it out using the p5.play demos. Read more details in the "Issue Deep-Dive" section below.

Then, I added the end condition. When the kitty sprite overlaps with any of the asteroids in the sprite group, the game ends and resets. With this, I finished the core functions of the game and had time to add more features. One of the additions was a timer, which I implemented using the setInterval() function. I also made the speed of the asteroids gradually increase to make the gameplay more interesting and challenging. For visual effects, I made the asteroids rotate and loaded an image of an asteroid icon to the sprites. Lastly, I made sure players can replay the game without having the refresh the entire page.

Issue Deep-Dive

The first issue I encountered was with the kitty avatar leaving a trail whenever the mouse moved. The code worked when I was using the p5,js editor but not when I wrote the same code on VSCode. I realized the reason was because I was adding to the starter code, which had the background() in the draw() function already. Meanwhile, I was accustomed to setting the background color in the setup() function. By having the background color set everytime the draw() function is looped, it covers the kitty image from the previous position and only shows the image where the mouse is currently positioned.

The second issue was with creating the asteroid sprites. Because I created the sprites in the draw() function, the asteroids were continually made until it covered and blocked the whole window. I changed it to create the sprites in the setup() functions. This made it so I could control the exact number of asteroids that would appear in the game. However, this meant I couldn't remove the asteroids once they left the window or else the asteroids will eventually all be removed, making it impossible to trigger the end condition of the game. So instead, I learned how to change the position of the asteroids so that once they moved past a certain number of pixels from the window edges, they would reappear on the other end of the window. Creating the asteroids in the setup() function also made it easier to reset the game without refreshing the window.
